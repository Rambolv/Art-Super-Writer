# 一、这是什么东西？

## 1.1 一句话

**超逸写手** 是一个专门为**写长篇小说**设计的AI辅助工具。它不只是一个AI聊天窗口，而是一整套写作工作台。

## 1.2 和直接聊AI的区别

| 场景 | 直接聊DeepSeek/ChatGPT | 用这个软件 |
|------|----------------------|-----------|
| **写第五章** | 需要把前四章全部粘贴给AI，一次粘不完，AI看到后面忘了前面 | **AI自动分批阅读全部已有章节**后再生成本章 |
| **角色一致性** | AI写错名字、死而复生、性格前后矛盾 | **自动注入角色档案**，审查官会检查连续性 |
| **质量检查** | AI说"写得很好"——没用的反馈 | **三个AI审查官**从13个维度打分，给具体修改建议 |
| **故事结构** | 写到哪算哪，前后矛盾不知道 | **世界树**显示主线/支线/伏笔的关系 |
| **节奏把控** | 凭感觉 | **节奏控制台**显示每章的吸引力评分曲线 |
| **写作上下文** | 手动复制粘贴前文 | **自动注入**：世界树+角色+节奏+写作建议 |

## 1.3 左侧导航栏总览

这个软件共有 **10 个页面**，通过左侧栏切换：

| 图标 | 页面 | 一句话说明 |
|------|------|-----------|
| 🎨 | **创作台** | 欢迎页+快速入口 |
| 📁 | **项目管理** | 创建/导入/锁定项目，执行AI分析 |
| ✍️ | **写作台** | **写小说的地方（核心功能）** |
| 🔍 | **审查室** | AI三审官检查章节质量 |
| 🌳 | **世界树** | 可视化故事结构（主线/支线/伏笔） |
| 🧠 | **角色池** | 每个角色的详细档案 |
| 📊 | **节奏控制台** | 各章节吸引力评分曲线 |
| ⚙️ | **设置** | LLM配置、分角色参数、质量门禁 |
| 📋 | **历史日志** | 查看AI分析的实时进度和完整记录 |
| 🤖 | **全自动长篇写作** | 开发中页面 |

# 二、下载和启动

## 2.1 下载

打开 https://github.com/Rambolv/Art-Super-Writer → 点击 **Code** → **Download ZIP** → 解压

## 2.2 安装Python

1. 打开 https://python.org/downloads
2. 下载 Python 3.10+，安装时**勾选「Add Python to PATH」**
3. 命令行输入 `python --version` 验证

## 2.3 安装依赖

```bash
cd 你的路径\standalone
python -m venv .venv
.venv\Scripts\activate
pip install flask requests
```

## 2.4 启动

**方法一：** 双击 `启动超逸写手.bat`
**方法二：** 双击 `超逸写手启动器.exe`（有界面的启动器）
**方法三：** `.venv\Scripts\python server.py` 然后浏览器打开 `http://127.0.0.1:8899`

# 三、申请 DeepSeek API Key

1. 打开 https://platform.deepseek.com/ → 注册 → 登录
2. 左侧菜单 **API Keys** → **Create API Key**
3. 取名 → 创建 → **复制 `sk-` 开头的密钥**（关掉就看不到了）

# 四、首次设置

## 4.1 添加LLM配置

左边栏点击 **⚙️ 设置** → 在「LLM供应商配置」点击 **➕ 添加LLM配置**：

| 字段 | 填什么 |
|------|--------|
| 名称 | 随便取，如"DeepSeek主用" |
| 供应商 | **DeepSeek** |
| 模型 | **`deepseek-chat`** |
| API Key | 粘贴刚才申请的 `sk-...` |
| 工作模式 | 选 **Stateless 模式** |

点击保存 → 测试连接 → 显示 **✅ 连接成功！** 即可。

## 4.2 配置分角色参数

不同功能可以用不同的AI配置。往下翻到「分角色参数配置」：

| 角色 | 温度 | 用途 |
|------|------|------|
| ✍️ **写作** | 0.8 | 生成小说，温度高更有创意 |
| 🔍 **分析** | 0.7 | 分析故事结构和角色 |
| 🔴 **严审** | 0.3 | 最严格的审查官 |
| 🟡 **衡审** | 0.3 | 平衡的审查官 |
| 🟢 **宽审** | 0.4 | 宽松审查官（读者视角） |
| 🗳️ **投票** | 0.2 | 最终裁决 |
| 📚 **技能学习** | 0.5 | 从分析中学习写作技巧 |

每个角色下拉菜单都选你刚创建的LLM，温度保持默认。点击底部 **💾 保存全部配置**。

# 五、两种方式开始你的第一本书

## 方式一：已有文档，导入后写作

### 第1步：🎨 创作台 → 新建项目

左边栏点击 **🎨 创作台** → 这是欢迎页面，可以在这里快速新建项目。或者点击 **📁 项目管理**。

> **创作台**（🎨）是软件的欢迎页，显示"欢迎使用 Art Super Writer"。如果你没有锁定项目，它也会提示你先锁定一个项目。快速入口可以直接在这里新建项目或打开最近项目。

### 第2步：创建项目

在 **📁 项目管理** 页面：
1. 输入**项目名称**（如"我的第一本书"）
2. 选择**小说类型**（玄幻、悬疑等，可选）
3. 可以写**故事前提**（一句话简介）
4. 点击 **✨ 创建**

### 第3步：导入文件

1. 在项目列表中点击你的项目展开详情
2. 找到 **「📥 导入章节文件」**，选择 `.txt` 或 `.md` 文件
3. 点击导入，导入后显示 **"1 章"**

### 第4步：锁定项目

点击项目右侧 **🔒 锁定** → 顶部显示 **已锁定「XXX」** → 此后续所有功能以此项目为核心

### 第5步：AI分析（8步流程）

点击 **🔍 分析** 按钮，AI自动执行：

| 步骤 | 做什么 |
|------|--------|
| ① 探测能力 | 问AI能处理多少内容 |
| ② 分批阅读 | 把你的全部分批发给AI读，每批回复"已读" |
| ③ 阅读体验 | 以读者身份评价吸引力/情感浓度 |
| ④ 故事结构 | 提取主线事件/支线/伏笔/世界观 |
| ⑤ 角色印象 | 识别所有角色，分析性格和行为模式 |
| ⑥ 整体感受 | 节奏评价+最想知道的三件事 |
| ⑦ 写作技能 | 编辑视角评价优缺点 |
| ⑧ 输出JSON | 生成结构化数据（世界树/角色/节奏） |

**分析期间不要刷新页面。** 可以点击 **📋 历史日志** 看实时进度。

### 第6步：查看分析结果

- **🌳 世界树** — 主线节点、支线状态、伏笔列表
- **🧠 角色池** — 每个角色的档案卡片
- **📊 节奏控制台** — 各章节吸引力评分

## 方式二：新开一本书，从头写

1. 创建项目（同上）
2. 锁定项目
3. 直接去写作台写第一章（新项目可以先不分析）
4. 写了几章后回项目管理点分析

# 六、写作台详解

## 6.1 界面布局

```
┌─────────────────────────────────┬──────────────────────┐
│          主面板（中间）         │    右侧面板          │
│                                 │                      │
│  📖 项目信息                    │  📖 项目统计         │
│                                 │  (章节数/角色数/是否分析)│
│  ┌──── 写作表单 ────────────┐   │                      │
│  │ 章节号: [ 1 ]             │   │  📦 前置SKILL        │
│  │ 写作方向:                  │   │  ☑ 对话规则          │
│  │ [文本框]                   │   │  ☑ 描写风格          │
│  │ 温度: 0.8 [===○====]      │   │  [✏️新建][📥导入]    │
│  │ 目标字数: [3000]           │   │                      │
│  │ [💾保存][📤发送给LLM]      │   │  📋 写作上下文       │
│  └─────────────────────────────┘   │  (自动注入的分析数据)  │
│                                 │                      │
│  📖 原文参考（可折叠）          │  📖 人类行为规律      │
│                                 │                      │
│  ┌── 生成结果 ──────────────┐   │                      │
│  │ [🔄重新生成][📌定稿][🔍分析]│  │                      │
│  │ [内容文本框-流式显示]      │   │                      │
│  │ ⚠️ 请等待生成完后编辑       │   │                      │
│  └─────────────────────────────┘   │                      │
└─────────────────────────────────┴──────────────────────┘
```

## 6.2 每个输入框

### 【章节号】
- 默认自动填下一章编号
- 可以手动改——重写某章时改成对应数字
- **切换章节号会自动加载该章的写作方向和草稿**

### 【写作方向】⚠️ 最重要

你告诉AI这一章要写什么。

✅ **好例子：**
```
主角张三到达古城后发现信中地点已被封锁。
他暗中打听得知最近发生多起离奇失踪案。
夜里他潜入封锁区，遇到一位同样在调查的女记者。
两人决定合作。
要求：悬疑氛围要足，对话要自然，为后续反转埋伏笔。
```

❌ **坏例子：**
```
写第四章
```

**可以包含的要素：** 关键事件、角色互动、氛围基调、特殊要求。

### 【温度】
- 0.1~0.5：稳定保守，适合严谨章节
- 0.6~0.9（推荐）：有创意不失控
- 1.0~1.5：有创意但可能跑偏

### 【目标字数】
- 短章1000~2000字（过渡章节）
- 标准2000~5000字（推荐）
- 超过5000字建议拆成两章

## 6.3 每个按钮

### 【💾 保存提示词】
- 把写作方向存到磁盘：`chapters/writer_direction_第001章.json`
- 切换章节时自动加载
- 不触发AI，纯保存

### 【📤 发送给LLM】核心按钮

点击后完整流程：

| 阶段 | 说明 | 耗时 |
|------|------|------|
| ① 前置检查 | 检查方向和LLM配置 | 瞬间 |
| ② 模式判断 | 判断当前LLM是Slot还是Stateless | 瞬间 |
| ③ 探测能力 | 问AI上下文上限 | 1~3秒 |
| ④ 模糊记忆 | 如全文太长，自动压缩旧章节 | 几秒~几十秒 |
| **⑤ 分批阅读** | **把你的全部已有章节分批发给AI读** | **10秒~几分钟** |
| ⑥ 写作指令 | 方向+上下文+SKILL+行为规律打包发AI | 瞬间 |
| ⑦ 流式生成 | AI逐字生成，实时显示在文本框 | 30秒~几分钟 |

**分批阅读是核心机制：** AI写新章前先通读你的所有已有章节，确保知道完整故事。

### 【🔄 重新生成】
- 不满意结果时点击重来
- 本质是重新点发送按钮

### 【📌 定稿】
- 把生成内容正式保存为章节
- 保存到 `chapters/chapter_XXX.md`
- **已定稿的章节再次定稿会弹出确认框**（防止误覆盖）

### 【🔍 分析本章】
- 仅分析当前这一章的质量
- 评分维度：读者吸引力、节奏匹配度、优缺点、改进建议

## 6.4 右侧面板

### 📦 前置SKILL

控制AI文风的关键功能。

- **新建SKILL：** 写写作规则，AI写时会遵守
- **导入SKILL：** 支持 `.txt` `.md` `.docx` `.xlsx` `.json`
- **勾选=激活**，不勾选=停用
- **SKILL示例：**
  ```
  标题：对话规则
  内容：对话要简短有力，一个角色一次说话不超过三句。
  用动作带出说话人，不要用"说道""回答道"。
  ```

### 📋 写作上下文（自动注入）

分析完成后自动生成的参考信息：

| 模块 | 注入内容 |
|------|---------|
| **故事进展** | 当前主线节点、读者最想知道的伏笔 |
| **角色行为** | 每个角色的性格、决策风格、在意/恐惧、关系网、一致性警告 |
| **阅读节奏** | 整体节奏评价、读者最想知道的问题 |
| **写作建议** | 本文优点（继续保持）、不足（注意改进） |
| **角色当前状态** | 每个角色所处位置、情绪、最近出场章节 |
| **已死亡角色** | 标记（非硬性限制，闪回/幻觉可出现） |
| **进行中的情节线** | 当前活跃支线的进度 |
| **待回收伏笔** | 埋下后尚未回收的伏笔 |

### 📖 人类行为规律参考

预设的行为心理学原则，AI写作时自动参考：
- 决策规律（损失厌恶、沉没成本等）
- 互动规律（信任博弈、公平需求等）
- 行为真实感规律（反常行为需铺垫等）

## 6.5 注意事项

- **生成过程中不要编辑文本框**——AI的后续输出可能乱掉
- 写方向就点 **保存提示词**，养成习惯
- 切换章节号自动加载方向和草稿
- 未定稿的内容自动保存草稿到 `drafts/`，切换页面不丢失
- 长文（100K+ tokens）的分批阅读可能需要几分钟，耐心等待

# 七、其他页面详解

## 7.1 🔍 审查室

**作用：** AI三审官检查章节质量，给修改建议。

### 操作步骤

1. 左边栏点击 **🔍 审查室**
2. 选择要审查的章节 → 点击 **🔍 审查**
3. 审查流水线自动执行：

```
⚙️ S1 机械闸（19条规则扫描）
   ├── 标点混用/破折号过多/省略号滥用
   ├── AI禁用句式（总结性短语、二元对比模板）
   ├── 段落长度均匀性、句子单一性
   ├── 主要角色缺席多章、情节线停滞
   └── 伏笔超期未回收、死角色活跃检查

🔴 S2 严审官（温度0.3，标准严格，逐项检查全部子项）
🟡 S2 衡审官（温度0.3，检查核心子项）
🟢 S2 宽审官（温度0.4，从大局评估，只标明显问题）

🗳️ 夹逼投票（三审取中值 → 优选最佳建议）

🚪 质量门禁
   ├── ≥75分：✅ 通过
   ├── 50~74分：🔄 修订
   └── <50分：⚠️ 人工审查
```

### 审查维度（13项）

| 维度 | 检查内容 |
|------|---------|
| 情节 | 逻辑自洽、因果关系、情节漏洞 |
| 角色 | 行为符合人设、动机清晰、成长弧线 |
| 对话 | 自然度、独特性、潜台词 |
| 节奏 | 场景匹配、高潮紧凑、章节钩子 |
| 文笔 | 句式变化、比喻新鲜、描写具体 |
| 一致性 | 时间线、角色细节、世界观规则 |
| 情感冲击 | 共情力、克制度、氛围营造 |
| 原创性 | 陈词滥调、套路化、可预测性 |
| 可读性 | 段落长度、句子结构、信息密度 |
| AI检测风险 | 段尾总结、过度解释、模板句式 |
| 角色心理 | 行为符合性格、决策体现欲望 |
| 社会互动 | 博弈逻辑、权力关系、信任机制 |
| 例外真实性 | 反常行为有铺垫、有代价、有后果 |

### 编辑器功能

- 审查后右侧显示**修改建议卡片**
- 点击卡片可**自动定位到原文位置**
- 支持 **撤销/重做**（Ctrl+Z / Ctrl+Y）
- 点击**应用**按钮自动替换原文

### 流水线指示器

审查过程中顶部显示进度条：
`S1 → 严审 → 衡审 → 宽审 → 选票 → 夹逼 → 投票 → 门禁`

## 7.2 🌳 世界树

**作用：** 可视化展示故事的结构和进展。

### 四个维度

| 维度 | 显示内容 | 帮你做什么 |
|------|---------|-----------|
| **主干**（主线） | 按顺序列出关键事件节点，标注重要性和所在章节 | 看清主线推进到哪了 |
| **枝丫**（支线） | 所有进行中的支线，标注当前进展和与主线关联度 | 知道哪些支线在跑 |
| **芽**（伏笔） | 按读者最想知道的迫切程度排序 | 知道哪些伏笔该收了 |
| **根系**（世界观） | 独特的设定和规则 | 保持世界观一致 |

### 使用方式

1. 分析完成后自动生成
2. 顶部有**版本选择器**，可切换查看不同分析版本的数据
3. 数据自动注入写作台，写新章时AI会参考

---

## 7.3 🧠 角色池

**作用：** 每个角色的详细档案卡片。

### 显示内容

每个角色显示：
- **角色名** + **定位**（主角/反派/配角）
- **读者标签**：读者对这个角色的印象
- **性格标签**：外向/谨慎/冲动...

# 八、高级概念

## 8.1 Slot 模式 vs Stateless 模式

**工作模式**在设置页为每个LLM配置单独指定。

| 对比 | Stateless 模式 | Slot 模式 |
|------|---------------|-----------|
| **适用场景** | DeepSeek、OpenAI等云端API | 本地 llama.cpp |
| **请求方式** | 每次发送完整对话历史 | 发送完整历史 + `slot_id` |
| **缓存机制** | 服务端可能自有缓存（无需前端管理） | llama.cpp通过`slot_id`管理KV缓存 |
| **多轮对话** | 每次请求携带全部历史，Token消耗大 | 服务端只处理新增消息，Token消耗小 |
| **配置方法** | 设置页工作模式选「Stateless」 | 设置页工作模式选「Slot」 |

**自动探测：** 连接成功后系统自动询问LLM支持哪种模式。也可手动指定。

## 8.2 模糊记忆（长文压缩）

**问题：** 当你的小说太长（超过AI的上下文窗口），AI读不完。

**解决方案：** 
1. 系统自动把旧章节压缩成摘要
2. 压缩后的摘要替代原文发送给AI
3. AI仍然知道整体故事，但损失一些细节
4. 压缩后的文件保存在 `fuzzyMemory/` 目录

**触发条件：** 全文Token数超过模型的建议输入大小时自动触发。

## 8.3 分析版本选择器

每次点击「分析」都会创建一个新版本（v1、v2、v3...）。

**在世界树/角色池/节奏控制台页面顶部**有版本选择器下拉菜单：
- 可以切换到任意历史版本查看
- 版本信息包含：分析时间、使用的LLM、是否压缩等

## 8.4 SKILL 的自动学习（skills/self/）

除了你手动创建的SKILL（`skills/external/`），AI还会**自动从分析结果中学习**。

- 每次分析完成后，AI的写作技能评价（第7轮）会被保存到 `skills/self/`
- 包括：本文优点（继续保持）、本文不足（注意改进）、综合建议
- 写作时自动注入到提示词中

## 8.5 中英文界面切换

页面顶部右上角有 **中 / EN** 按钮，点击切换界面语言。

## 8.6 面板宽度调整

中间面板和右侧面板之间有一条**可拖拽的分隔条**，鼠标按住可以左右拖动调整面板宽度。

# 十一、常见问题

## Q1: AI文风不对怎么调？
用SKILL。写作台右侧「📦 前置SKILL」新建规则，勾选激活即可。

## Q2: 分析结果不满意？
顶部版本选择器切换到旧版本查看。或重新分析生成新版本。

## Q3: 能同时写多本书吗？
可以。解锁→锁定其他项目，数据完全独立。

## Q4: API Key安全吗？
本地加密存储+绑定机器指纹，其他电脑无法解密。

## Q5: 需要联网吗？
分析/写作/审查需要联网（调AI API）。查看历史数据可离线。

## Q6: 支持哪些AI模型？
DeepSeek、OpenAI兼容API、本地llama.cpp、任何兼容OpenAI格式的模型。

## Q7: 本地LLM怎么用？
设置中添加LLM→供应商选"自托管"→地址填 `http://127.0.0.1:8080`→模式选Slot或Stateless→测试连接。

## Q8: 写了一半切页面会丢吗？
不会。自动保存草稿到 `drafts/`，回来自动加载。

## Q9: 页面崩溃了？
刷新即可。所有数据在磁盘上，定稿的章节、分析结果、SKILL都不丢。

## Q10: 怎么导出？
`projects/你的项目/chapters/` 下的 `.md` 文件直接复制即可。

## Q11: Slot和Stateless选哪个？
云端API（DeepSeek/OpenAI）选Stateless。本地llama.cpp选Slot。

## Q12: 什么是分批阅读？
AI写新章前，先把你的全部已有章节切成小块，一批一批发给AI读完。这样AI知道完整故事后再写新章，不会前后矛盾。

## Q13: 什么是模糊记忆？
当全文太长超过AI容量时，自动把旧章节压缩成摘要，节省空间给新内容。

## Q14: 为什么发送后要等很久？
分批阅读需要时间。全文越长越久，100K tokens约需1~3分钟。耐心等待，可在「历史日志」看进度。

## Q15: 提示词保存了但切换章节没加载？
检查服务器是否在运行。方向文件保存在 `chapters/writer_direction_第001章.json`，需要服务器API读写。
