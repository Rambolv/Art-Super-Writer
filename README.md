# 🚀 它好在哪里？/ Why This?

## 👍 优势

| 你头疼的问题 | 超逸写手的解法 |
|-------------|--------------|
| **AI 写了后面前面忘了**，要反复粘贴前文 | **自动分批阅读全文**——写新章前 AI 先通读你所有章节 |
| **角色写崩了**：名字写错、性格飘忽、死人复活 | 🧠 **角色池**跟踪每个角色的性格/关系/状态，写作时自动注入提示词 |
| **写完不知道好不好**，问 AI 它只说"写得不错" | 🔍 **三审流水线**——严/衡/宽三个 AI 审查官从 13 个维度打分 |
| **故事结构松散**，写到后面发现前面挖的坑忘了填 | 🌳 **世界树可视化**——主线/支线/伏笔一目了然 |
| **节奏把控不好**，该紧张的地方不紧张 | 📊 **节奏控制台**——逐章吸引力评分曲线 |
| **每次写新章要手动整理前文**发给 AI | ✍️ **写作上下文自动注入**——世界树+角色+节奏+写作建议 |

## 👎 劣势（诚实地说）

| 方面 | 说明 |
|------|------|
| **需要 API Key** | 不能免费白嫖，需要申请 DeepSeek/OpenAI 账号并充值 |
| **需要一点动手能力** | 安装 Python 和依赖，大概 10 分钟搞定 |
| **不能离线使用** | 写作和分析需要调用云端 AI（但查看历史数据可以离线） |
| **不是点一下自动出一本书** | 它是辅助工具，不是自动生成器——你仍然需要构思和编辑 |
| **UI 比较朴素** | 单 HTML 文件，功能优先于颜值 |

> **一句话：** 如果你在认真写长篇小说，它能帮你省大量重复劳动；如果你想点一下就自动生成一本完整小说，它不适合你。

# ⚡ 30 秒快速开始 / Quick Start

## 1️⃣ 装 Python

打开 [python.org](https://python.org/downloads) → 下载 Python 3.10+ → 安装时**勾选 "Add Python to PATH"**

## 2️⃣ 装这个软件

```bash
# 下载后进入目录
cd 你解压的路径\standalone

# 创建虚拟环境（只要做一次）
python -m venv .venv

# 激活虚拟环境（每次运行前都要做）
.venv\Scripts\activate

# 安装依赖（只要做一次）
pip install flask requests
```

> ⏳ 国内用户如果下载慢：`pip install flask requests -i https://pypi.tuna.tsinghua.edu.cn/simple`

## 3️⃣ 启动

```bash
# 双击也行
python server.py
```

浏览器打开 **http://127.0.0.1:8899** → 看到界面 → 完成 🎉

> 💡 Windows 用户也可以直接双击 `启动超逸写手.bat` 或 `超逸写手启动器.exe`

## 4️⃣ 获取 API Key（必须）

1. 打开 [platform.deepseek.com](https://platform.deepseek.com) → 注册 → 登录
2. 左侧菜单 **API Keys** → **Create API Key** → 取名 → 创建
3. **复制 `sk-` 开头的密钥**（关掉窗口就看不到了）

## 5️⃣ 配置

在软件界面左边栏点击 **⚙️ 设置**：

1. **添加 LLM 配置** → 填写：
   - 供应商：`DeepSeek`
   - 模型：`deepseek-chat`
   - API Key：粘贴刚才复制的 `sk-...`
   - 工作模式：`Stateless`
2. 点击 **保存** → **测试连接** → ✅ 连接成功

# 📖 详细使用指南

## 📁 项目管理——从这里开始

### 新建项目
输入项目名称 → 选择小说类型 → 点击「✨ 创建」

### 导入已有文档
点击项目 → 「📥 导入章节文件」→ 选择 `.txt` 或 `.md` → 导入

### 锁定项目
点击 **🔒 锁定** → 锁定后才能使用写作台/审查室等功能

### AI 分析
点击 **🔍 分析** → AI 执行 8 步分析（不要刷新页面）：

```
① 探测模型能力 → ② 分批阅读全文 → ③ 阅读体验
→ ④ 故事结构 → ⑤ 角色印象 → ⑥ 整体感受
→ ⑦ 技能评价 → ⑧ 输出 JSON
```

完成后数据自动保存到世界树/角色池/节奏控制台。

> 每次写完新章后建议重新分析，让下一章的写作拥有最新上下文。

## ✍️ 写作台——核心战场

### 写作流程

```
写方向 → 💾 保存 → 📤 发送给 LLM → ✏️ 编辑 → 📌 定稿
```

### 完整执行流程（点击发送后）

| 阶段 | 说明 | 耗时 |
|------|------|------|
| ① 探测能力 | 问 AI 上下文上限 | 1~3 秒 |
| ② 分批阅读 | **把你的所有已有章节分批发给 AI 读完** | 10 秒~几分钟 |
| ③ 注入上下文 | 自动拼入：世界树+角色档案+节奏+SKILL+行为规律 | 瞬间 |
| ④ 流式生成 | AI 逐字生成，实时显示在文本框 | 30 秒~几分钟 |

### 写作方向怎么写？

✅ **好的：**
```
主角张三到达古城后发现信中地点已被封锁。
他暗中打听，得知最近发生多起离奇失踪案。
夜里潜入封锁区，遇到一位同样在调查的女记者。
要求：悬疑氛围，对话自然，为后续反转埋伏笔。
```

❌ **不好的：**
```
写第四章
```

越具体越好——包含：关键事件、角色互动、氛围基调、特殊要求。

### 右侧面板是什么？

| 面板 | 作用 |
|------|------|
| 📦 **前置 SKILL** | 你写的写作规则，AI 写时遵守。可新建/导入/勾选激活 |
| 📋 **写作上下文** | 自动注入的分析数据（世界树+角色+节奏） |
| 📖 **行为规律** | 预设的心理学原则，AI 参考用 |

### 📦 SKILL 示例
```
标题：对话规则
内容：对话要简短有力，一个角色一次不超过三句。
用动作带出说话人，不要用"说道""回答道"。
```
保存后勾选即可激活。支持导入 `.txt/.md/.docx/.xlsx/.json`。

## 🔍 审查室——质量检查

选择章节 → 点击 **🔍 审查** → 自动执行：

```
S1 机械闸（19 条规则扫描）
  → 检查标点/AI句式/段落/连续性
  
S2 三审官（13 个维度评分）
  → 🔴 严审(温度0.3) 🟡 衡审(温度0.3) 🟢 宽审(温度0.4)
  
🗳️ 夹逼投票 → 🚪 质量门禁
  → ≥75分通过 | 50~74分修订 | <50分人工
```

修改建议以卡片形式显示在右侧，点击可自动定位原文并应用修改。编辑器支持撤销/重做。

## 🌳 世界树——看清你的故事

| 维度 | 含义 |
|------|------|
| **主干** | 主线剧情节点，标注重要性和所在章节 |
| **枝丫** | 支线进展，标注与主线的关联度 |
| **芽** | 伏笔，按读者最想知道的迫切程度排序 |
| **根系** | 世界观设定和独特规则 |

顶部有**版本选择器**，可切换查看不同分析版本的数据。

---

## 🧠 角色池——角色档案

每个角色显示：定位、性格标签、行为模式、决策风格、人际关系、一致性警告。

- 点击角色卡片展开/折叠详情
- 右侧可编写**自定义分析提示词**进行针对性分析
- 顶部版本选择器切换不同版本

---

## 📊 节奏控制台——把控阅读体验

- **逐章评分**：每章的吸引力(1-10)和情感浓度(1-10)
- **作用标签**：引入/铺垫/冲突/转折/高潮...
- **连续性健康度**：角色缺席、支线停滞、伏笔超期等警告

# ⚙️ 设置说明

## LLM 工作模式：Slot vs Stateless

| | Stateless | Slot |
|--|-----------|------|
| **适用** | DeepSeek、OpenAI 等云端 API | 本地 llama.cpp |
| **原理** | 每次请求发完整对话历史 | 发完整历史 + slot_id，服务端管理 KV 缓存 |
| **推荐** | ✅ 大多数用户选这个 | 只有用本地 llama.cpp 时才选 |

## 分角色参数

每个角色可独立指定不同的 LLM 配置：

| 角色 | 推荐温度 | 用途 |
|------|---------|------|
| ✍️ 写作 | 0.8 | 生成小说内容 |
| 🔍 分析 | 0.7 | 分析故事结构 |
| 🔴 严审 | 0.3 | 严格审查 |
| 🟡 衡审 | 0.3 | 平衡审查 |
| 🟢 宽审 | 0.4 | 宽松审查 |
| 🗳️ 投票 | 0.2 | 最终裁决 |
| 📚 技能学习 | 0.5 | 学习写作技巧 |

# ❓ 常见问题 / FAQ

**Q: 需要会写代码吗？**
A: 不需要。但需要按教程装 Python（跟着做就行，约 10 分钟）。

**Q: 要花多少钱？**
A: DeepSeek 新用户有免费额度，用完后按量计费。写一本几十万字的小说大约几块钱。

**Q: 能同时写多本书吗？**
A: 可以。解锁→锁定不同项目即可，数据完全独立。

**Q: AI 写的文风不对怎么办？**
A: 在写作台右侧「📦 前置 SKILL」中新建写作规则，AI 会遵守。

**Q: 写了一半切页面会丢吗？**
A: 不会。草稿自动保存到 `drafts/`，回来自动加载。

**Q: 页面崩溃了怎么办？**
A: 刷新即可。所有数据在磁盘上，不会丢。

**Q: 支持哪些 AI？**
A: DeepSeek（推荐）、OpenAI 兼容 API、本地 llama.cpp。

**Q: 什么是分批阅读？**
A: AI 写新章前先把你的全部已有章节读完——这样它知道完整故事，不会前后矛盾。
