# 一、这是什么东西？和直接让AI写有什么区别？

## 1.1 这是个什么软件

**超逸写手** 是一个专门为**写长篇小说**设计的工具。它不只是一个AI聊天窗口，而是一整套**写作工作台**。

你可以把它理解为一个**写作工作室**，里面有：
- **一个写作台**——写新章节的地方
- **一个分析团队**——读完你的全文后告诉你故事结构、角色问题
- **三个审查官**——严审、衡审、宽审，从不同角度挑毛病
- **一个世界树展示板**——主线/支线/伏笔的关系图
- **一个角色池**——每个角色的性格档案
- **一个节奏控制台**——每章的吸引力评分

## 1.2 和直接打开DeepSeek网页聊天有什么区别

| 场景 | 直接聊AI | 用这个软件 |
|------|---------|-----------|
| **让AI写第一章** | 告诉AI设定，AI写出来，还行 | 同样告诉设定，AI写出来，也还行 |
| **让AI写第五章** | 你需要把前四章全部粘贴给AI，一次粘不完，分多次粘，AI看到后面忘了前面 | **AI自动分批阅读全部已有章节**，你只需要说"写第五章" |
| **角色前后一致** | AI把主角名字写错、前面死的角色后面又活了、性格前后矛盾 | **自动注入角色档案**到提示词里，审查官还会检查连续性 |
| **写完想检查质量** | 让AI读一遍，AI说"写得很好"，没什么用 | **三个AI审查官**分别从严格/平衡/宽松角度打分，给出具体修改建议 |
| **想看清故事结构** | 让AI总结，AI说一堆废话 | **世界树可视化**：主线节点、支线进度、伏笔回收状态一目了然 |
| **把握节奏** | 凭感觉 | **节奏控制台**显示每章的吸引力评分曲线 |
| **写下一章时参考前文** | 手动复制粘贴 | **自动注入**世界树+角色+节奏数据到写作提示词 |

**核心区别一句话：** AI聊天是一次性的——每次对话都是独立的。这个软件是有**记忆**和**分析能力**的——它读完你的全部章节后，会持续跟踪角色、情节、节奏，在写作时自动给你提供上下文。

# 二、下载和启动

## 2.1 下载

**方法一：下载ZIP包（推荐新手）**

1. 打开网页：https://github.com/Rambolv/Art-Super-Writer
2. 点击绿色的 **「Code」** 按钮
3. 点击 **「Download ZIP」**
4. 解压到电脑上的一个文件夹（建议路径不要有中文，比如 `D:\Writer\`）

**方法二：用Git下载（如果你会Git）**

```
git clone https://github.com/Rambolv/Art-Super-Writer.git
```

## 2.2 安装Python

> 这个软件需要 Python 来运行后端服务器。如果你电脑上还没有Python，需要先安装。

1. 打开 https://python.org/downloads
2. 点击黄色的 **Download Python 3.10+** 按钮
3. 下载完成后双击安装
4. **重要：** 安装时务必勾选 **「Add Python to PATH」**（把Python添加到系统路径）
5. 点击 **Install Now** 完成安装

**验证安装：**
1. 按 `Win + R`，输入 `cmd`，回车
2. 输入 `python --version`
3. 如果显示 `Python 3.10.x` 或更高版本，说明安装成功

## 2.3 首次安装依赖

打开命令行（cmd），进入软件目录：

```bash
cd D:\Writer\Art-Super-Writer-main\standalone
python -m venv .venv
.venv\Scripts\activate
pip install flask requests
```

> 如果下载慢，可以使用国内镜像：
> `pip install flask requests -i https://pypi.tuna.tsinghua.edu.cn/simple`

## 2.4 启动软件

**方法一：双击 `启动超逸写手.bat`（最简单）**

找到 `standalone` 文件夹，双击 **`启动超逸写手.bat`**。
会弹出一个黑窗口，等待显示服务器启动完成。
打开浏览器，地址栏输入：`http://127.0.0.1:8899`

**方法二：手动启动**

双击 `start_server.bat`，浏览器打开 `http://127.0.0.1:8899`

## 2.5 常见启动问题

**问：黑窗口一闪就没了？**
答：说明启动出错了。用记事本打开 `start_server.bat`，在最后一行 `pause` 前加上 `pause`（已有了就不用加），再双击运行，黑窗口会停住显示错误信息。最常见的原因是Python没装或者虚拟环境没创建。

**问：浏览器打开后显示空白？**
答：看黑窗口还在不在。如果黑窗口还在，试试按 `F5` 刷新页面。如果黑窗口关闭了，重新启动。

**问：端口被占用？**
答：关掉其他可能占用8899端口的程序，或者重启电脑。

# 三、申请 DeepSeek API Key（获取AI能力）

> 这个软件需要调用AI来分析和写作。你需要一个 **API Key**——相当于你使用AI的"钥匙"。
> 新注册用户有免费额度，够写几本书。用完后按量计费，写一本几十万字的小说大约几块钱。

6. **立刻复制它，保存到记事本里**（关掉窗口就看不到了，只能重新创建一个新的）

# 四、首次设置

> 第一次打开软件，你需要告诉它用哪个AI。

## 4.1 进入设置页

左边栏点击 **⚙️ 设置**

## 4.2 添加LLM配置

在「LLM供应商配置」区域：
1. 点击 **「➕ 添加LLM配置」**
2. 弹出一个表单，填写：
   - **名称**：随便取，比如"DeepSeek主用"
   - **供应商**：选择 **DeepSeek**
   - **模型**：填 **`deepseek-chat`**
   - **API Key**：粘贴你刚才申请的以 `sk-` 开头的密钥
   - **工作模式**：选择 **Stateless 模式**
3. 点击 **「💾 保存」**
4. 点击 **「🔌 测试连接」**
5. 如果显示 **「✅ 连接成功！」**，说明配置正确

> **什么是模型名？** AI公司有多个版本的AI，deepseek-chat 是 DeepSeek 目前最强的模型。

## 4.3 配置分角色参数

**这一步很重要**，不同功能可以用不同的AI配置。往下翻到「分角色参数配置」：

| 角色 | 温度 | 做什么的 |
|------|------|---------|
| ✍️ **写作** | 0.8 | 生成小说内容——温度高更有创意 |
| 🔍 **分析** | 0.7 | 分析故事结构、角色 |
| 🔴 **严审** | 0.3 | 最严格的审查官 |
| 🟡 **衡审** | 0.3 | 平衡的审查官 |
| 🟢 **宽审** | 0.4 | 宽松的审查官（读者视角） |
| 🗳️ **投票** | 0.2 | 最终裁决 |
| 📚 **技能学习** | 0.5 | 从分析中学习写作技巧 |

每个角色左边的下拉菜单都选择你刚创建的 **"DeepSeek主用"**。
温度不用改，保持默认就行。

> **什么是温度？** AI生成时的"创意程度"。0.1=很保守很稳定，1.5=很有创意但可能跑偏。写作用0.8比较合适。

## 4.4 保存设置

点击页面底部的 **「💾 保存全部配置」**。

# 五、两种方式开始你的第一本书

## 方式一：你已经有写好的文档，想导入进来

**适合：** 你已经在记事本/Word里写了一部分小说。

### 第1步：新建项目

1. 左边栏点击 **📁 项目管理**
2. 在「新建项目」区域，输入**项目名称**（比如"我的第一本书"）
3. 选择**小说类型**（玄幻、悬疑、言情等）
4. **故事前提**可以填一句话介绍，也可以先空着
5. 点击 **✨ 创建**

### 第2步：导入文件

1. 项目创建后，在下方项目列表中找到它
2. 点击项目卡片展开详情
3. 找到 **「📥 导入章节文件」** 区域
4. 点击选择文件 → 选你电脑上的 `.txt` 或 `.md` 文件
5. 点击导入
6. 上方卡片会显示 **"1 章"**

### 第3步：锁定项目

1. 在项目列表中，点击右侧的 **🔒 锁定** 按钮
2. 页面顶部会显示 **已锁定「我的第一本书」**
3. **必须锁定才能使用写作台、审查室等功能**

### 第4步：AI分析

1. 锁定后，在项目详情中点击 **🔍 分析**
2. AI开始执行8步分析（不要刷新页面）：

| 步骤 | 说明 |
|------|------|
| 第1步 | AI自报上下文能力 |
| 第2步 | 分批阅读你的全部章节 |
| 第3步 | 评价阅读体验和吸引力 |
| 第4步 | 分析故事结构（主线/支线/伏笔） |
| 第5步 | 分析每个角色的性格和行为 |
| 第6步 | 评价节奏和读者感受 |
| 第7步 | 评价写作优缺点 |
| 第8步 | 输出JSON结构化数据 |

3. 完成后提示 **✅ 分析完成！**
4. 过程中可以点 **「📋 历史日志」** 看实时进度

### 第5步：查看分析结果

- **🌳 世界树** — 主线节点、支线进度、伏笔列表
- **🧠 角色池** — 每个角色的性格、行为模式、人际关系
- **📊 节奏控制台** — 各章节的吸引力评分

## 6.2 每个输入框详解

### 【章节号】 `writerChNum`

- 默认自动填下一个章节号（比如已有3章，默认填4）
- 可以手动修改——比如你想重写第2章，就改成2
- 修改后会自动加载该章节已保存的**写作方向**和**草稿**

### 【写作方向】 `writerDirection`

**这是整个软件最重要的输入框。** 你在这里告诉AI这一章要写什么。

**好的写法：**
```
主角张三到达古城后发现信件中提到的地方已经被封锁。
他暗中打听，得知最近古城里发生了多起离奇失踪案。
夜里他偷偷潜入封锁区，遇到了一个同样在调查此事的女记者。
两人决定合作。要求：
- 悬疑氛围要足，环境描写烘托紧张感
- 女记者的形象要鲜明，对话要自然
- 为后续的情节反转埋下伏笔
```

**不好的写法：**
```
写第四章
```

**为什么越具体越好？** AI不是你肚子里的蛔虫，你不说清楚它就会自由发挥。方向写得越具体，生成的结果越接近你的预期。

你可以写的要素：
- **关键事件**：这一章主要发生什么事
- **角色互动**：谁和谁见面、发生什么冲突或合作
- **氛围基调**：紧张、温馨、悬疑、悲伤...
- **特殊要求**：避免什么、强调什么、埋什么伏笔

### 【温度】 `writerTemp`

滑动条，范围 0.1 ~ 1.5：
- 0.1~0.5：非常稳定，AI基本按常规套路写，适合需要严谨控制的章节
- 0.6~0.9（推荐）：有创意但不失控，适合大多数情况
- 1.0~1.5：非常有创意，但可能跑偏，适合需要脑洞的章节

**新手建议** 保持在 0.8。

### 【目标字数】 `writerWords`

告诉AI生成多少字。输入框默认3000字，可调范围500~20000。
- 短章（1000~2000字）：适合过渡章节
- 标准章（2000~5000字）：推荐，太长AI可能质量下降
- 长章（5000+字）：AI可能会啰嗦，建议拆成两章

## 6.3 左侧按钮详解

### 【💾 保存提示词】 `btnSavePrompt`

**作用：** 把你写的「写作方向」保存到磁盘，下次切换章节再切回来还能看到。

**什么时候用？** 每次写完方向就点一下保存，养成习惯。
**保存到哪里？** `chapters/writer_direction_第001章.json`（跟章节文件放一起）

**注意：**
- 只保存「写作方向」文本框里的内容，不触发AI
- 切换章节号时会自动加载该章节已保存的方向
- 服务器没启动时不能保存（会静默失败）

### 【📤 发送给LLM】 `btnWrite`

**这是核心按钮——开始写作。**

**点击后会发生什么（完整流程）：**

| 阶段 | 说明 | 耗时 |
|------|------|------|
| ①前置检查 | 检查是否有方向文本、LLM是否配置 | 瞬间 |
| ②模式判断 | 检测当前LLM是Slot模式还是Stateless模式 | 瞬间 |
| ③第1步：探测能力 | 发送"你的上下文上限是多少？"给AI | 1~3秒 |
| ④模糊记忆 | 如果全文很长，先压缩旧章节 | 几秒~几十秒 |
| **⑤第2步：分批阅读** | **把你所有已有章节分批发给AI读，每批回复"已读"** | **10秒~几分钟** |
| ⑥第3步：写作指令 | 把写作方向+上下文+SKILL+行为规律打包发给AI | 瞬间 |
| ⑦生成内容 | AI开始流式生成小说内容，**实时显示在文本框** | 30秒~几分钟 |

**重点理解第5步（分批阅读）：**
这是这个软件跟直接聊AI最大的区别。当你已经有前几章的内容时，AI写新章之前会：
1. 把你的所有章节切成合适大小的批次
2. 一批一批发给AI阅读
3. AI每读完一批回复"已读"
4. 全部读完后才开始写新章

**这样AI就知道了全部前文，不会出现前后矛盾。**

**按钮状态：**
- 点击后按钮变成 **「⏳ 发送中...」** 并禁用
- 生成完成后按钮恢复为 **「📤 发送给LLM」**
- 如果LLM配置不对，会提示错误并自动恢复按钮

**注意：**
- 如果全文太长（100K+ tokens），分批阅读可能需要几分钟
- 这是正常现象，耐心等待
- 可以在「历史日志」中查看实时进度

## 6.4 生成结果区域的按钮

### 【🔄 重新生成】 `btnRegenerate`

不满意当前结果？点击它重新生成一遍。
本质上是重新触发了「发送给LLM」按钮，流程同上。

**注意：**
- 重新生成不会保留前一次生成的结果
- 如果前一次结果里有你喜欢的段落，先复制到别处

### 【📌 定稿】 `btnFinalize`

**作用：** 把生成的内容正式保存为一章。

**点击后：**
1. 检查内容是否为空
2. 检查该章节是否已经定稿过
3. **如果已经定稿过**，弹出确认框：
   `⚠️ 该章节(第X章)已经定稿，再次定稿会覆盖原有内容，请确认是否覆盖？`
   点击确定→覆盖，取消→不操作
4. 保存到内存 `p.chapters`
5. 写入磁盘 `chapters/chapter_XXX.md`
6. 如果是 Slot 模式，关闭当前 LLM session

**注意：**
- 定稿后内容会出现在「已有章节」列表中
- 定稿后可以继续修改，再次定稿会覆盖
- 定稿不等于最终版——你随时可以重写

### 【🔍 分析本章】 `btnSaveChapter`

**作用：** 对刚生成的这一章做质量分析。

**分析维度：**
- 读者吸引力（1-10分）
- 节奏匹配度（1-10分）
- 优点列表
- 不足列表
- 改进建议
- 综合评分

**注意：**
- 这个分析是**独立的**，只分析这一章，不看全文
- 如果想看全文分析，用「项目管理」中的「分析」按钮
- Slot模式下会复用写作session（如果有），Stateless模式独立请求

## 6.5 生成结果文本框 `writerContent`

- AI生成的内容会**实时流式显示**在这里——一个字一个字地出现
- 生成完成后你可以**直接在这个文本框里编辑修改**
- 橙色提示：**「请等待生成结果全部完成后再编辑文本」**
- 如果你在生成过程中修改，AI的后续输出可能会乱掉，建议等完成后再改

## 6.6 右侧面板详解

### 📖 项目统计

- 显示当前项目共多少章、多少个角色、是否已分析

### 📦 前置SKILL

**这是控制AI文风的关键功能。**

SKILL 是你写给AI看的**写作规则**。你可以创建多个SKILL，勾选=激活，AI写的时候会自动参考。

**创建SKILL：**
1. 点击 **✏️ 新建**
2. 标题：比如"对话规则"
3. 内容：比如"对话要简短有力，一个角色一次说话不超过三句。不要用'说道''回答道'，直接用动作带出说话人。"
4. 保存

**导入SKILL：**
1. 点击 **📥 导入**
2. 选择 `.txt`、`.md`、`.docx`、`.xlsx`、`.json` 文件
3. 自动提取内容并保存为SKILL

**勾选/取消：**
- 勾选（绿色）= 激活，写作时AI会参考
- 取消勾选（灰色）= 停用，AI不看这个

**删除：** 点右边的 ✕，会二次确认

**SKILL的注入位置：**
在发送给AI的提示词中，SKILL内容放在系统提示词（system prompt）里，位于行为规律之前、写作指令之前。所以AI会把它当作**必须遵守的规则**。

**建议：**
- 新建项目时先创建几个基础SKILL（比如你的文风偏好、常用描写手法）
- 随着写作推进不断优化SKILL内容
- 可以创建多个SKILL并自由开关

### 📋 写作上下文

这是AI写作时的**参考信息**，自动从你的分析结果中提取，包括：
- **故事进展**：当前主线推进到什么节点
- **角色行为模型**：每个角色的性格、决策风格、在意/害怕的事
- **节奏数据**：当前节奏评价、读者最想知道的问题

你不需要手动操作，这些数据会自动展示在这里，并注入到AI的提示词中。

### 📖 人类行为规律参考

这是预设的**写作心理学原则**，写作时AI会自动参考。包括：
- 决策规律（损失厌恶、沉没成本、参照点依赖等）
- 互动规律（信任博弈、公平需求等）
- 行为真实感规律（反常行为需要的铺垫等）

这些原则会注入到写作提示词中，让AI生成的角色行为更真实可信。

# 七、写作完成之后

## 7.1 审查

见下一节「八、审查室」。

## 7.2 重新分析

写完一章并定稿后：
1. 回到 **📁 项目管理**
2. 点击 **🔄 重新分析**
3. AI重新读取所有章节（含新写的），更新世界树和角色池

**为什么要重新分析？** 这样下一章写作时，注入的上下文就是最新的——包含了新章节的角色发展、情节进展、伏笔信息。

## 7.3 日常写作循环

```
写新章 → 审查 → 定稿 → 重新分析 → 继续写下一章...
```

每次循环大约需要：
- 写一章：2~5分钟（AI生成）+ 你自己编辑的时间
- 审查：1~3分钟
- 重新分析：2~5分钟（取决于全文长度）

# 八、审查室

审查室的作用：在AI帮你写的章节里找出问题。

## 8.1 如何审查

1. 左边栏点击 **🔍 审查室**
2. 选择要审查的章节
3. 点击 **🔍 审查**
4. 审查自动执行以下流水线：

```
⚙️ S1 机械闸（规则检查）
   ├── 半角标点混入中文
   ├── 破折号过多、省略号滥用
   ├── AI禁用句式（总结性短语、二元对比模板等）
   ├── 段落长度是否均匀
   ├── 主要角色是否缺席多章
   ├── 情节线是否停滞
   └── 伏笔是否超期未回收

🔴 S2 严审官（温度0.3，标准严格）
🟡 S2 衡审官（温度0.3，标准适中）
🟢 S2 宽审官（温度0.4，标准宽松）

🗳️ 夹逼投票（三审意见汇总+优选）

🚪 质量门禁
   ├── ≥75分：通过
   ├── 50~74分：修订
   └── <50分：人工审查
```

## 8.2 如何应用修改

审查完成后，每条修改建议会以卡片形式显示在右侧：
- 点击建议卡片，编辑器会自动定位到原文位置
- 点击 **应用** 按钮自动替换原文
- 编辑器支持撤销/重做（Ctrl+Z/Ctrl+Y）

# 十、常见问题

## Q1: AI写出来的文风不对，怎么办？

两种方法：

**方法一：用SKILL控制（推荐）**
在写作台右侧「📦 前置SKILL」中创建规则。例如：

```
标题：描写风格
内容：描写环境时多用比喻和通感。
对话要简洁，用动作带出说话人而不是"他说""她说"。
心理描写要克制，用行为暗示情绪而不是直接说"他很伤心"。
```

保存并勾选，下次AI写时会遵守。

**方法二：调整温度**
温度调低（0.5~0.7）AI更稳定保守，调高（0.9~1.2）更有创意。

## Q2: 分析结果不满意？

每次分析都会创建新版本。在世界树/角色池/节奏页面顶部有版本选择器，可以切换到之前版本查看。

## Q3: 能同时写多本书吗？

可以。在「项目管理」中解锁当前项目，再锁定另一个项目。不同项目的数据完全独立。

## Q4: API Key安全吗？

保存在你的电脑本地硬盘，用加密方式存储，并绑定了电脑硬件信息。即使有人拿到你的文件，在其他电脑上也无法解密。

## Q5: 需要一直联网吗？

分析、写作、审查时需要联网（调用AI的API）。查看之前的历史数据可以离线。

## Q6: 支持哪些AI模型？

- DeepSeek（推荐，性价比最高）
- 任何兼容 OpenAI API 格式的服务（GPT-4、Claude 等）
- 本地 llama.cpp（如果配置了 Slot 模式）

## Q7: 本地 LLM 怎么用？

1. 在设置中添加LLM配置，供应商选择「自托管」
2. 填写你的本地地址：`http://127.0.0.1:8080`
3. 工作模式选「Slot模式」（如果支持）或「Stateless模式」
4. 测试连接

## Q8: 写了一半切到别的页面，内容会丢吗？

不会。未定稿的内容会自动保存到 `drafts/` 文件夹。切换页面再回来，草稿会自动加载到文本框。

## Q9: 页面崩溃了怎么办？

所有数据都保存在磁盘上。刷新页面即可恢复，已定稿的章节、分析结果、SKILL都不会丢。

## Q10: 写好的小说怎么导出？

章节文件在 `projects/你的项目名/chapters/` 目录下，直接复制 `.md` 文件即可。也可以用记事本打开，复制内容到Word。
