# 🎯 优势和劣势

## 它解决了什么问题

| 你遇到的麻烦 | 超逸写手怎么解决 |
|-------------|----------------|
| **AI 记不住前文**——聊着聊着忘了前几章写了什么，角色性格乱变 | **自动分批阅读全文**——写每一章之前，AI 先把你的全部已有章节通读一遍 |
| **角色前后矛盾**——名字写错、性格飘忽、死了的人又复活 | 🧠 **角色档案自动注入**——每个角色的性格/关系/状态被记录下来，写作时自动放到提示词里 |
| **写完不知道好不好**——让 AI 评价，它只会说"写得很好" | 🔍 **三个 AI 审查官**——严审/衡审/宽审从 13 个维度打分，给出具体修改建议 |
| **故事结构失控**——挖了坑忘了填，支线写丢了 | 🌳 **世界树**——主线节点/支线进度/伏笔状态一目了然 |
| **节奏感差**——该紧张的地方平淡，该收尾的地方拖沓 | 📊 **节奏控制台**——逐章吸引力评分+连续性健康检查 |
| **花费高**——每次写作都要把全文发给云端 AI，Token 费用飞涨 | 💰 **Slot 模式 + 本地 LLM**——用 llama.cpp 的 slot 缓存，同一段上下文只处理一次，费用最低可降到日均几分钱 |

## 诚实地说：不足在哪里

| 不足 | 说明 |
|------|------|
| 需要一点动手能力 | 装 Python + 装依赖，大约 10 分钟 |
| 写作和分析需要联网 | 但如果用本地 LLM，全程可以不依赖云端 |
| 不是全自动写书机 | 你仍然需要构思剧情、编辑生成结果——它是**辅助**，不是替代 |
| UI 功能优先 | 单 HTML 文件实现，没做花哨动画 |

## 关于省钱

**你可以不花一分钱。** 这款软件设计时就考虑了两种运行模式：

| 方案 | 成本 | 适合 |
|------|------|------|
| **本地 LLM（Slot 模式）** | 💰 **几乎免费**（只有电费） | 你有 8G+ 显存的显卡 |
| **云端 API（DeepSeek）** | 写一本几十万字约几块钱 | 你不想折腾本地部署 |

> 作者强烈推荐本地 LLM + Slot 模式：llama.cpp 的 slot KV 缓存会让 AI 读完一次全文后只需处理新增消息，**一遍全文+连续写多章，上下文只预热一次**。这是整个软件最核心的省钱设计。

# ⚡ 30 秒上手 · 30 Seconds Quick Start

### 第一步：装 Python

→ [python.org](https://python.org/downloads) 下载 Python 3.10+，安装时 **勾选 Add to PATH**

### 第二步：装依赖

```bash
cd 你的路径\standalone
python -m venv .venv
.venv\Scripts\activate
pip install flask requests
```

### 第三步：启动

双击 `启动超逸写手.bat`（或 `.venv\Scripts\python server.py`）→ 浏览器打开 `http://127.0.0.1:8899`

### 第四步：连接 AI

选一种：

**方案 A——云端 API（推荐新手）**
去 [platform.deepseek.com](https://platform.deepseek.com) 注册 → API Keys → 创建一个 Key → 复制 `sk-...`。在软件设置里添加 LLM 配置，粘贴 Key。

**方案 B——本地 LLM（推荐有显卡的用户，最省钱）**
见下方 [详细使用指南 C](#c-用本地-llm最省钱的方案)。

# 📂 分情况详细指南

## A. 你已有文档，想继续写

**场景：** 你之前用 Word/记事本写了部分小说。

### ① 新建项目

左边栏点 **📁 项目管理** → 输入项目名 → 选小说类型 → **「✨ 创建」**

### ② 导入文件

点击项目 → **「📥 导入章节文件」** → 选 `.txt` 或 `.md` → 导入

### ③ 锁定项目

点 **🔒 锁定** → 锁定后写作台/审查室等模块都以此项目为核心工作

### ④ AI 分析

点 **🔍 分析** → AI 执行 8 步分析（别刷新页面）：
```
探测能力 → 分批阅读全文 → 阅读体验 → 故事结构
→ 角色印象 → 整体感受 → 技能评价 → 输出 JSON
```
完成后世界树/角色池/节奏数据自动生成。

### ⑤ 查看数据

- **🌳 世界树**：主线到哪了、支线在跑什么、伏笔收了没
- **🧠 角色池**：每个角色的档案卡片
- **📊 节奏控制台**：各章评分

### ⑥ 开始写作

去 **✍️ 写作台**，填写作方向 → 发送 → 等 AI 生成 → 编辑 → 定稿。

写完整章后回项目管理**重新分析**，更新数据后再写下一章。

---

## B. 新开一本书，从零写

### ① 新建 + 锁定

**📁 项目管理** → 创建 → 锁定

### ② 开始写第一章

去写作台直接写（新项目没有前文可以跳过分析）。

### ③ 写几章后分析

有 2~3 章后回项目管理点分析。之后每写一章重新分析一次。

## C. 用本地 LLM（最省钱的方案）

**前提：** 你有一块 8G+ 显存的 NVIDIA 显卡。

### ① 安装 llama.cpp

从 [llama.cpp GitHub](https://github.com/ggerganov/llama.cpp/releases) 下载 Windows 版本，解压到 `LLM` 文件夹（放在 `standalone/LLM/`）。

### ② 下载模型

推荐 Qwen2.5 7B/14B 中文模型（HuggingFace 可下载），放到 `LLM/models/`。

### ③ 配置启动脚本

在 `LLM/` 里创建 `start.bat`：

```bat
llama-server.exe -m models/qwen2.5-7b.gguf --host 127.0.0.1 --port 8080 --ctx-size 65536 -ngl 99 --slot-save-path slots/
```

关键参数：
- `--ctx-size 65536` = 64K 上下文（越大越能一次读完你的全文）
- `-ngl 99` = 把 99 层都放到 GPU（充分发挥显卡性能）
- `--slot-save-path` = 持久化 slot 缓存（重启后还能复用预热好的上下文）

### ④ 在设置中配置

**⚙️ 设置** → 添加 LLM 配置：

| 字段 | 填什么 |
|------|--------|
| 供应商 | 自托管 |
| API地址 | `http://127.0.0.1:8080` |
| 模型 | `qwen2.5-7b` |
| API Key | 留空（本地不需要） |
| **工作模式** | ⚡ **Slot** |

> ⚡ **Slot 模式是核心。** 它让 llama.cpp 通过 `slot_id` 管理 KV 缓存。全文分批阅读时，每批的缓存被保留；写下一章时读取全文的缓存直接复用——上下文只预热一次，后续每次写新章只需追加新增消息。如果不用 Slot，同一个上下文每次都要完整处理，这就是为什么云端 API Token 费用高的原因。

### ⑤ 测试连接

点测试连接 → ✅ 成功 → 回到分角色参数，把写作/分析/审查都指定为这个本地 LLM → 保存。

## 📦 前置 SKILL——控制文风

SKILL 是你写给 AI 的写作规则。创建后勾选=激活。

```
标题：对话规则
```

# ⚙️ 配置详解

## LLM 工作模式

| 模式 | 原理 | 什么时候用 |
|------|------|-----------|
| ⚡ **Slot**（推荐） | llama.cpp 通过 slot_id 管理 KV 缓存，全文预热一次后续只发新增消息 | 本地 llama.cpp |
| 🔵 Stateless | 标准 API，每次请求发完整对话历史 | DeepSeek、OpenAI 等云端 API |

**自动探测：** 连接成功后系统自动检测是 Slot 还是 Stateless。也可以在设置里手动指定。

## 分角色参数

每个功能可独立指定 LLM 和温度：

| 角色 | 温度 | 说明 |
|------|------|------|
| ✍️ 写作 | 0.8 | 生成小说内容——温度高更有创意 |
| 🔍 分析 | 0.7 | 分析故事结构——温度适中 |
| 🔴 严审 | 0.3 | 严格审查——温度低更严谨 |
| 🟡 衡审 | 0.3 | 平衡审查 |
| 🟢 宽审 | 0.4 | 宽松审查（读者视角） |
| 🗳️ 投票 | 0.2 | 夹逼投票裁决——温度最低最稳定 |
| 📚 技能学习 | 0.5 | 学习文风规律 |

## 质量门禁

审查时使用的通过标准：

| 参数 | 默认 | 说明 |
|------|------|------|
| 通过阈值 | 75 | 总分≥75 直接通过 |
| 修订阈值 | 50 | 总分≥50 需修订 |
| 最大迭代 | 3 | 自动修订最多循环次数 |
| 维度最低分 | 5 | 任一维度低于此分 +2 分修正 |

## 审查权重 & 维度权重

三审官评分的加权系数和各维度的权重可独立调整，在设置页下半部分配置。

# ❓ 常见问题

**Q: 必须花钱吗？**
A: 不用。本地部署 llama.cpp + Slot 模式是最省钱的方案，全程不花 API 费用。如果不方便本地部署，DeepSeek 新用户有免费额度，用完后写一本几十万字也只要几块钱。

**Q: 需要会编程吗？**
A: 不需要。按教程装 Python 和敲几行命令就行。如果本地部署 LLM 需要稍微多一点操作，但也不涉及写代码。

**Q: AI 写的风格不对怎么办？**
A: 写作台右侧 📦 前置 SKILL，新建写作规则，AI 会自动遵守。

**Q: 写了一半切页面会丢吗？**
A: 不会。自动保存草稿到 `drafts/`，切换回来自动加载。

**Q: 崩溃了怎么办？**
A: 刷新即可。所有数据在磁盘上，不会丢。

**Q: 支持什么 AI？**
A: DeepSeek、OpenAI、本地 llama.cpp、任何兼容 OpenAI 格式的 API。

**Q: 能同时写几本书？**
A: 可以。解锁→锁定其他项目，数据独立。

**Q: 什么是分批阅读？**
A: 写新章前 AI 把你的全部章节切成小块、一批一批读完。读完才知道完整故事，写出来的新章才不会跟前文矛盾。
