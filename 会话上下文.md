### 关键设计原则（已确立）

1. **LLM以读者身份分析** — 不是文学专家，是刚读完小说的普通读者
2. **分析结果回流注入Writer** — 世界树/角色池/节奏数据在写作时注入提示词
3. **写作一次写好** — 不依赖多次审查修正
4. **展示和注入并行** — 页面展示给用户看，同时数据喂给Writer
5. **LLM自行归类** — 不是前端解析JSON字段，是LLM自己输出结构化数据
6. **OCEAN/DarkTriad不作为强制框架** — 读者自然描述即可

### 数据流向

```
写作台 → 章节 → 读者分析(analyzeProject)
                   ├→ worldTree(剧情结构) → 世界树页面 + Writer注入
                   ├→ characters(角色印象) → 角色池页面 + Writer注入
                   ├→ rhythm(阅读节奏) → 节奏控制台页面 + Writer注入
                   └→ memo(写作备忘录) → 下一章写作时注入
```
