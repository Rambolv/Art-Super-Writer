    · 收敛度 c = 1/(1+w)
    · 有界中心 bc = (U+L)/2
    · 中值对齐度 a = 1 - |M(衡) - bc|

    · 完美<0.15 / 紧<0.30 / 中<0.50 / 松≥0.50
  → VoterAgent(置信度<85%时):
    · 评估4维度: 风格对齐度+情节一致性+改进潜力+风险
    · temp=0.2(低温度确定性投票)
  → Resolver(平局时):
    · 裁决优先级: 共识度>审查官权重>严重程度>优先级分数

Phase 5: 应用最优建议
  → WriterAgent.revise(original, [best_suggestion], story_context, style_guide)

Phase 6: 质量门禁循环(最多3次)
  → s2_gate.re_evaluate() → 重新评分
  → QualityGate.evaluate():

    · <50 或 严审任一维度<5: FAIL — 需人工介入
    · 3次仍不过: NEEDS_HUMAN
  → 每次迭代记录到 review_history

  → 保存final章节到 chapters/
  → 同步到 {项目名}资料/文件夹
  → 保存审查总结到 {项目名}资料/第N章_审查总结.md

### 五、S1 反AI-Slop规则库（完整版）

**标点规则(3条):**
- `[,.;:!?]` 半角标点 → BLOCK，修复: 替换为全角
- `——.*——.*——` 单句3+破折号 → BLOCK
- `\.\.\..*\.\.\.` 单段省略号>2次 → WARN

**禁用句式(7条):**
- `不是[^，,\n]{1,30}[，,而是]` "不是X而是Y"二元对比 → BLOCK
- `[。！？\n]\s*(其实|老实说|不得不说|讲真|说真的|平心而论|坦白说)` 句首填充词 → BLOCK
- `(总之|总而言之|综上所述|由此可见|不难看出|不难发现|这告诉我们)` 段尾总结 → BLOCK
- `(这(意味着|说明|表明|是因为|暗示着|代表着))` 过度解释 → WARN
- `[？?]\s*(答案|原因|道理|关键)...` 自问自答 → WARN
- `(就像|仿佛|宛如|犹如|如同).{1,20}(一样|一般|似的)` 强制比喻 → WARN
- `(感到|觉得|感觉|认为|想道|心想|暗想)` 情感标签过度 → WARN

**结构问题(5条):**
- 对话标签单调: 连续5次"说/道"无变化
- 句子长度单一: 连续4句长度差异<20%
- 段落长度均匀: AI特征每段3-4句
- 关键词重复: 同一词3句内>2次
- 被动语态密度>15%

**连续性引擎(7条):**
- 休眠情节线: >3章未推进 → WARNING
- 超期情节线: 超过目标解决章节 → CRITICAL
- 未回收伏笔: >10章无回收 → WARNING
- 角色缺席: 主要角色>5章未出现 → WARNING
- 未登场角色: 已定义未出现 → CRITICAL
- 死亡角色活跃: 已死亡仍有状态 → WARNING
- 时间线漂移: 推进与章节不匹配 → CRITICAL

## 世界树 (World Tree) 是什么

世界树是整个故事**情节结构的有向生长模型**，把故事映射为一棵树：

| 组件 | 含义 | 例子 |
|------|------|------|
| 🌲 **主干 (Trunk)** | 主线剧情节拍 — 主角的关键行动 | "第3章: 林星辰发现远古遗迹" |
| 🌿 **枝丫 (Branch)** | 支线/角色弧的发展 | "艾琳娜调查家族秘密" |
| 🌱 **芽 (Bud)** | 埋下的伏笔 | "暗影领主可能是林星辰的父亲" |
| 🍂 **叶 (Leaf)** | 已解决的情节/已回收的伏笔 | "凯恩博士的过去在第6章揭晓" |
| 🌍 **根 (Root)** | 世界观设定 | "星际联邦成立于2387年" |

**核心机制**：重要性指数衰减（半衰期10章）— 枝丫离主线越远，重要性越低，达到阈值就会触发"自然提醒"。

## 模糊记忆 (Fuzzy Memory) 是什么

模糊记忆是**一个引擎容器**，里面装了 5 个子模块：

```
模糊记忆 (Fuzzy Memory Engine)
 ├── 🌲 世界树 (WorldTreeManager)    ← 情节结构
 ├── 🪢 伏笔追踪 (ForeshadowingTracker) ← U形曲线
 ├── 🔔 自然提醒 (NaturalReminder)   ← L0-L5提醒策略
 ├── 📊 张力波 (TensionWaveManager)  ← 节奏控制

```

## 两者的关系

**模糊记忆是容器，世界树是容器里的一个子模块。** 书写成后，编排器 Orchestrator 调用 `generate_fuzzy_memory_context()`，这个方法：

```
1. 世界树生长 → grow_chapter(chapter_num)
2. 取世界树摘要 → get_growth_summary()
3. 取提醒指导 → get_reminder_guidance()  ← 枝丫离主线太远？
4. 取伏笔上下文 → get_foreshadowing_context() ← 哪些芽没开花？
5. 伏笔追踪分析 → get_callback_guidance() ← U形曲线
6. 张力波分析 → get_wave_context() ← 现在该蓄力还是释放？
7. 是否需要压缩 → should_compress() ← 上下文快满了？
     ↓
  全部拼接 → 注入 Writer 的提示词 → AI写作时自然使用
```

**一句话总结**：世界树是"情节结构数据库"，模糊记忆是"聚合+提炼这些数据库并注入Writer的引擎"。世界树只负责存和算，模糊记忆负责把世界树+伏笔+张力+提醒整合成一段AI能用的上下文。

# ASR Writer HTML5 独立版 — 需求分析书（草稿）

## 一、系统定位

AI 辅助长篇小说写作系统的前端独立版。单 HTML5 文件运行，数据本地存储，模拟完整流水线：写作 → S1审查 → S2三审查官 → 夹逼定理投票 → 质量门禁。

## 二、功能模块

### 模块1：项目管理
用户可创建项目（名称+前提+类型），导入 .md/.txt 小说文件自动识别分章，选择/删除项目，查看项目概览（章节数、角色数、评分趋势）。

### 模块2：写作工作室
用户输入章节方向，调节温度/字数等参数，模拟生成章节内容。完成后显示章节列表（含评审状态和评分），支持"继续写下一章"和"前往审查"。

### 模块3：审查仪表盘
展示9阶段流水线指示器，S1反AI-Slop扫描结果列表，三审查官并排展示（13维度评分条），各维度夹逼定理数据分析表（U/M/L值、宽度、收敛度、质量分），投票裁决结果（置信度、裁决理由），质量趋势折线图。

### 模块4：世界树
展示主干（主线剧情节拍）、枝丫（支线+角色弧）、芽（伏笔）、叶（已解决）、根（世界观）。含健康度进度条、重要性指数衰减展示、伏笔总览（未回收/已回收）。

### 模块5：心理学实验室
角色OCEAN大五人格、Dark Triad暗黑三角、道德基础配置滑条。切换标签页查看行为经济学决策模拟、纳什均衡博弈论分析、例外验证σ级别检测、角色对比表。支持手动创建角色。

### 模块6：张力波
张力曲线折线图（含4分区基线），五阶段张力写作指导表（平静/蓄力/上升/高潮/回落），当前张力/趋势/近3章均值统计。

### 模块7：设置
10个LLM供应商及对应模型选择，API Key分供应商存储。质量门禁阈值配置（通过/修订/迭代次数）。审查权重配置（严/衡/宽），13维度权重配置。反AI-Slop规则库展示。

### 模块8：历史日志
按时间线展示输入/输出记录，支持按章节筛选。

**问题是：纯 HTML5 在浏览器中无法创建磁盘文件夹。** 这是浏览器安全限制，没有变通方法。

解决这个问题的实际方案有几个：

1. **保持虚拟文件夹** — 所有数据在 localStorage 里，磁盘上不建文件夹。轻量、便携、无需安装
2. **加一个简单的本地服务器** — 比如用 Python 一行命令启动，同时提供 API 创建文件夹
3. **Electron 桌面应用** — 用 Node.js 做后端，可以读写磁盘

您倾向哪种？如果选方案2，只需要在项目根目录跑 `python -m http.server` 或者直接复用已有的 Python 环境，我可以在 HTML 里通过 fetch 调用后端的文件创建接口。
