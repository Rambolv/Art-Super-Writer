## 项目概述

基于原 ASR Writer（AI 辅助长篇小说写作系统）的核心设计，提取为独立运行的 HTML5 单页应用 + Python 后端。

### 技术栈
- **前端**: 纯 HTML5 + CSS3 + Vanilla JS（单页应用）
- **后端**: Python 3 + http.server（用于磁盘文件夹创建）
- **图表**: Chart.js v4.4.7（质量趋势图）
- **LLM**: 浏览器直接调用 DeepSeek API

## 核心设计原则

### LLM 分析流程
```
所有章节 → 估算Token
  ├─ ≤55K → 全部原文发送
  └─ >55K → 旧章L3压缩(400字摘要) + 最近5章保留原文
       ↓
以读者身份阅读 → 自然讲述感受 → 自行归类为结构化数据
       ↓
存入项目 → 展示到各页面 + 下一章写作时注入Writer
```

### 分析Prompt结构
1. **角色定位**: 刚读完小说的普通读者
2. **逐章体验**: 吸引力/情感浓度/作用标签
3. **故事结构**: 主线/支线/伏笔/世界观
4. **角色印象**: 特点/在意/害怕/关系/一致性
5. **整体感受**: 节奏判断/最想知道/下一章期待/提醒
6. **结构化输出**: LLM自己归类为 JSON

### 模块关系
```
写作台 (生产者) ──章节──→ 读者分析 (一次调用)
                              ├→ 世界树 (剧情结构)
                              ├→ 角色池 (角色印象)
                              ├→ 节奏控制台 (阅读体验)
                              └→ 写作备忘录 (下一章注入)
```

### 关键区分
- **世界树** = 精确剧情数据库（无损，AI直接写入）
- **模糊记忆** = 上下文窗口兜底（有损压缩，L0→L4）

## 配置详情

### LLM 供应商
| 供应商 | API Key 环境变量 | 默认模型 |
|--------|-----------------|---------|
| DeepSeek | DEEPSEEK_API_KEY | deepseek-v4-flash |
| OpenAI | OPENAI_API_KEY | gpt-4o |
| 🛠️ 自定义 | CUSTOM_API_KEY | 自定义 |

### 质量门禁默认值
- 通过阈值: 75（推荐70-80）
- 修订阈值: 50（推荐45-55）
- 最大迭代: 3（推荐3）
- 严审最低维度分: 5（推荐5）

### 审查权重默认值
- 严审: 1.0（推荐1.0）
- 衡审: 0.7（推荐0.7）
- 宽审: 0.4（推荐0.4）

### 13维度权重
全部默认为 1.0（情节/角色/对话/节奏/文风/一致性/情感冲击/原创性/可读性/AI味检测/角色心理/社会互动/例外真实性）
